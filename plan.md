# 음압패치 효과 검증 대시보드 — 구현 계획

## 목표

**핵심 질문: "음압패치를 함께 사용하면 얼마나 더 효과적인가?"**

14개 임상시험 PDF(휴먼테스트 수행)에서 시험군(제품+패치) vs 대조군(제품 단독) 데이터를 파싱하여, 패치의 부가 효과를 지표별/제품별로 비교하는 대시보드.

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| **프레임워크** | Streamlit | Python 단일 파일로 대시보드 완성, 설치 즉시 실행 |
| **PDF 파싱** | PyMuPDF (fitz) | 이미 설치됨, 한글/영문 추출 검증 완료 |
| **데이터 처리** | pandas | 테이블 정형화, 피벗, 통계 |
| **차트** | Plotly | 인터랙티브, 호버 시 상세 수치 표시 |
| **데이터 저장** | JSON | 파싱 결과 캐싱, 재파싱 불필요 |

## 파일 구조

```
├── data/                  # PDF 원본 (기존)

│   ├── app.py             # Streamlit 대시보드 메인

└── requirements.txt       # streamlit, plotly, pymupdf, pandas
```

## 대시보드 화면 구성

### 섹션 1. 상단 — Hero KPI 카드

패치 부가효과 핵심 수치를 카드로 한눈에 표시.
전 지표 군간비교 p<0.05 유의 표시.

### 섹션 2. 지표별 패치 효과 — 그룹 막대 차트

시험군(제품+패치) vs 대조군(제품 단독) 개선율 비교.
사이드바 필터로 보고서/지표 선택.
호버 시 Mean±SD, p-value, 대조군 대비 개선율 표시.

### 섹션 3. 제품×지표 히트맵

행: 제품, 열: 지표, 셀: 대조군 대비 개선율(%).
색상 스케일: 낮음(노랑) → 높음(빨강).
셀 클릭 시 해당 보고서 상세.

### 섹션 4. 피험자별 Raw Data — 산점도 + 박스플롯

개인별 시험군 vs 대조군 개선율 분포.
드롭다운으로 지표 선택.
각 점 호버 시 피험자 No, 연령, 개선율 표시.
N.A. / -†(개선율 0% 이하) 자동 제외, 탈락자 수 표시.

### 섹션 5. 흡수도 특별 섹션

패치 유/무에 따른 흡수량/깊이/속도 비교 (3건 통합).
뺨 vs 전박 부위별 비교 탭.
예비시험(n=3) 라벨 표시.

### 섹션 6. 안전성 요약

전 제품 이상반응 0건, 자극지수 0.07~0.18 (전 제품 비자극성), 순응도 99.3~100%, 설문 만족도 85~100%.

## 새 PDF 추가 방식

### 기본: 자동 감지 (A)

1. 새 PDF를 `data/` 폴더에 복사
2. Streamlit 앱 새로고침
3. 파서가 `parsed_data.json`에 없는 새 PDF를 감지 → 자동 파싱 → JSON 업데이트

### 폴백: 수동 입력 (B)

- 파싱 실패 시 웹 UI에서 "데이터 추가" 폼 제공
- 보고서명, 제품, 지표별 수치를 직접 입력 → JSON 저장

```
PDF 추가 → 자동 파싱 시도
              ├── 성공 → 대시보드 반영
              └── 실패 → "파싱 실패" 알림 + 수동 입력 폼 안내
```

## 구현 순서

| Phase | 작업 | 산출물 |
|-------|------|--------|
| **1** | PDF 파싱 엔진 — 요약 테이블 추출 | `parser.py`, `parsed_data.json` |
| **2** | PDF 파싱 엔진 — raw data 추출 | `parsed_raw_data.json` |
| **3** | 대시보드 — 섹션 1~3 (KPI, 막대, 히트맵) | `app.py`, `charts.py` |
| **4** | 대시보드 — 섹션 4~6 (산점도, 흡수도, 안전성) | `app.py` 확장 |

## Raw Data 파싱 참고

### 별첨 구조 (모든 보고서 공통)

- 피험자 정보: No, 이니셜, 성별, 연령
- 지표별 원시값: No.01~23 × 시험군/대조군 × 시점별
- 개인별 개선율(%): 시험군/대조군 각각
- 순응도: 사용횟수/순응도(%)
- 설문: 피험자별 6점 척도 응답

### 주의사항

- **N.A.**: 탈락자 데이터 → 제외 처리
- **-†**: 개선율 0% 이하 → 0 또는 음수로 처리
- 페이지 넘김 시 테이블 분할 → 연속 페이지 합치기 필요
- Windows cp949 인코딩 → stdout UTF-8 래핑 필요
